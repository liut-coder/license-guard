# License Guard 产品说明书

## 1. 产品概述

License Guard 是一个面向软件厂商的授权与完整性管理平台，用于给 Windows、Android 等客户端应用提供授权激活、设备绑定、完整性验证、风险事件分析和后台运营能力。

首期产品聚焦 Windows Go App 接入，目标是帮助现有 Windows 应用快速具备：

- 授权码激活
- 设备绑定
- 短期授权 token
- 基础完整性上报
- 风险事件可视化
- 授权吊销与设备封禁

## 2. 目标用户

### 2.1 平台管理员

负责系统配置、管理员账号、全局安全策略、审计日志。

主要需求：

- 管理后台登录安全。
- 查看所有应用和风险概况。
- 管理管理员权限。
- 审计敏感操作。

### 2.2 产品/运营人员

负责应用、授权套餐、客户授权和设备状态。

主要需求：

- 创建应用。
- 签发、续期、吊销授权。
- 查看客户设备绑定情况。
- 处理换设备、授权超限、误封申诉。

### 2.3 安全/研发人员

负责 SDK 接入、完整性配置、风险规则、异常分析。

主要需求：

- 查看 SDK 接入参数。
- 配置版本签名和 hash。
- 查看篡改、调试、Hook、重放等风险事件。
- 调整不同应用的风控动作。

### 2.4 外部应用开发者

负责把 License Guard SDK 接入到 Windows Go App。

主要需求：

- 获得 App ID、Public Key、API Endpoint。
- 调用激活、验证、心跳接口。
- 根据授权结果控制功能开关。
- 处理授权失败和离线场景。

## 3. 产品定位

一句话定位：

> 面向多应用的软件授权与完整性控制台。

设计调性：

## 4. 核心概念

### 4.1 应用 App

被保护的软件产品。一个 App 可以有多个平台配置，例如 Windows、Android。

字段：

- 应用名称
- App ID
- 平台
- 当前版本
- 环境
- 授权策略
- 完整性策略
- SDK Key

### 4.2 授权 License

代表客户或用户的使用权。

授权类型：

- 授权码
- 账号订阅
- 企业席位
- 试用授权

状态：

- 有效
- 试用
- 暂停
- 已吊销
- 已过期

### 4.3 设备 Device

一个安装并激活 App 的终端。

状态：

- 正常
- 待复核
- 已封禁
- 已解绑

### 4.4 激活 Activation

License 与 Device 的绑定关系。

状态：

- active
- blocked
- deactivated

### 4.5 完整性 Integrity

客户端上报的运行环境和二进制状态。

Windows 首期采集：

- App 版本
- 主程序 hash
- 签名证书指纹
- OS 版本
- 调试器状态
- 可疑模块
- 虚拟机信号

### 4.6 风险事件 Risk Event

服务端根据验证请求、完整性报告和规则生成的异常记录。

事件类型：

- 签名不匹配
- 二进制 hash 不匹配
- Debugger attached
- Hook/注入模块
- 设备数超限
- token 重放
- token 与设备不匹配
- 离线授权超期
- 短时间频繁换设备

## 7. 核心业务流程

### 7.1 创建应用

```text
管理员进入应用管理
点击新增应用
填写应用名称、平台、App ID、环境
系统生成 Public Key 和 SDK 配置
管理员进入应用详情配置授权策略
```

### 7.2 签发授权

```text
运营进入授权管理
点击签发授权
选择应用、授权计划、设备数量、到期时间
系统生成 license key
客户拿 license key 激活客户端
```

### 7.3 Windows Go App 激活

```text
用户输入 license key
App 请求 challenge
App 采集设备和完整性信息
App 调用 activate
服务端校验授权、设备数、完整性和风险
服务端绑定设备并返回短期 token
App 缓存 token 并解锁功能
```

### 7.4 常规启动验证

```text
App 启动
读取本地 token
token 未过期时进入基础可用状态
后台异步 verify
服务端返回新 token 或拒绝原因
App 更新授权状态
```

### 7.5 风险处置

```text
服务端发现异常
生成 risk_event
按规则执行 allow / challenge / deny / revoke / review
后台展示事件
管理员复核并处理
```

## 9. 权限设计

角色：

- Owner：全部权限。
- Admin：应用、授权、设备、风险管理。
- Operator：授权和设备管理。
- Security：风险事件和完整性配置。
- Viewer：只读。

敏感操作：

- 查看 SDK Secret
- 轮换 SDK Key
- 吊销授权
- 封禁设备
- 修改全局安全策略
- 删除或归档应用

敏感操作必须写入审计日志。

## 10. 首期验收标准

产品验收：

- 能创建 Windows 应用。
- 能生成 SDK 接入参数。
- 能签发 license。
- Windows Go Demo 能激活授权。
- 后台能看到设备、验证记录和风险事件。
- 能封禁设备，封禁后客户端无法获取新 token。

体验验收：

- UI 风格符合 Apple-like 高端简洁要求。
- 后台无默认紫色主色。
- 卡片、按钮、输入框、Badge 风格统一。
- 桌面和移动宽度不出现文字重叠。

安全验收：

- 客户端不包含服务端私钥。
- token 是短期有效。
- license key 不以明文保存。
- 管理后台敏感操作有审计日志。
- 服务端能防止 challenge/token 重放。
